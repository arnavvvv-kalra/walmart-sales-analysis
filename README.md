# walmart-sales-analysis
Leveraged Python for data cleaning and exploratory analysis, and executed advanced SQL queries to solve complex business problems for Walmart. Extracted key performance metrics to identify the top 3 categories by revenue, busiest days, and peak business hours to drive data-backed strategy.

## Tools & Technologies Used
- Python (Pandas)
- MySQL
- SQL Queries
- Jupyter

## Project Workflow

### 1. Data Cleaning & Preprocessing (Python - Pandas)
- Imported and explored the raw Walmart sales dataset.
- Performed data cleaning to improve data quality and consistency.
- Handled missing values and duplicate records.

### 2. Data Analysis & Business Problem Solving (SQL)
Loaded the cleaned dataset into MySQL and wrote SQL queries to answer complex business questions, including:

- Identified the busiest days of the week based on sales transactions.
- Analyzed peak sales hours to determine the time period with maximum customer activity.
- Found the top 3 product categories generating the highest revenue.
- Analyzed branch-wise sales performance and customer ratings.
- Calculated key business metrics such as total revenue, profit margin, and sales trends.
- Studied customer purchasing patterns across different product categories.

## Key Insights Generated
- Determined high-performing product categories contributing significantly to revenue.
- Identified peak transaction periods to understand customer buying behavior.
- Compared sales performance across different branches.

## Conclusion
This project demonstrates the use of Python for data preprocessing and SQL for analytical problem-solving to transform raw sales data into actionable business insights.-
